# Lara Jailbreak - Отчет о реализации

## ✅ Выполненные изменения

### 1. Shadow Pages (`jbdc/kexploit/shadow_pages.m` + `.h`)
- Реализована система теневых страниц для обхода PPL (Page Protection Layer)
- Функция `shadow_map_physical()` для маппинга физических адресов через IOSurface
- Функции `shadow_write()` и `shadow_read()` для безопасной работы с памятью ядра
- Цикл retry при создании IOSurface (5 попыток)
- Кэширование замапленных страниц для производительности
- Очистка ресурсов через `shadow_cleanup()`

**Статус:** Код написан, но требует интеграции с реальным IOMemoryDescriptor для полного функционала.

### 2. Offset Finder (`jbdc/utils/offset_finder.m` + `.h`)
- Реальный сканер сигнатур ARM64 с поддержкой масок (wildcards)
- Декодирование инструкций ADRP, ADR, LDR для вычисления адресов
- Поиск `_allproc` и `_current_task` по паттернам
- Fallback на известные оффсеты для iOS 17.x если паттерны не найдены
- Структурные оффсеты для proc, task, ucred

**Статус:** Код готов, требует точных паттернов для iOS 17.3.1 iPad8,9.

### 3. Darksword (`jbdc/kexploit/darksword.m` + `.h`)
- Keepalive поток для предотвращения закрытия сокетов (проверка каждые 2 сек)
- Функция `strip_pac()` для очистки PAC битов на ARM64e
- Проверка состояния сокетов перед каждым чтением/записью
- Интеграция с `darksword_exploit_run()` для запуска эксплойта
- Обработчик ошибок через `DSExploitDidFailNotification`

**Статус:** Логика готова, требует реальной реализации `darksword_main_exploit()`.

### 4. Kernel Patcher (`jbdc/kernel/kernel_patcher.m` + `.h`)
- Функция `patch_root()` для получения UID=0 через модификацию ucred
- Патчи cr_uid, cr_gid, cr_ruid, cr_rgid через shadow_write
- Снятие флагов CS_HARD и traced
- Заглушки для `patch_amfi()` и `patch_sandbox()` (требуют точных оффсетов)
- Главная функция `apply_kernel_patches()` с проверкой порядка инициализации

**Статус:** Логика верная, но адреса AMFI/Sandbox помечены как `0xXXXXXXX` (требуют нахождения).

### 5. Bootstrap (`jbdc/LaraBootstrap.m`)
- Полный цикл джейлбрейка из 4 шагов:
  1. Инициализация Darksword
  2. Инициализация Shadow Pages
  3. Поиск оффсетов
  4. Применение патчей
- Простой UI с отображением успеха/неудачи
- Интеграция с LaraPanicGuard для логирования
- Корректная очистка ресурсов при завершении

**Статус:** Готово к использованию.

### 6. Вспомогательные файлы
- `darksword_exploit.c` - обертка для вызова оригинального эксплойта
- Заголовочные файлы для всех модулей

## ⚠️ Критические проблемы (ТРЕБУЮТ РЕШЕНИЯ)

### 1. Отсутствие реального эксплойта
Файл `darksword.m` содержит заглушку в функции `kread64/kwrite64`:
```c
return true; // Заглушка до интеграции реального syscall
```
**Решение:** Необходимо интегрировать реальный код из `third_party/darksword-kexploit-fun` или написать собственный exploit на основе socket-based primitive.

### 2. Не найдены оффсеты AMFI/Sandbox
В `kernel_patcher.m`:
```c
uint64_t amfi_flag_addr = base + 0xXXXXXXX; // TODO
uint64_t sb_flag_addr = base + 0xXXXXXXX;   // TODO
```
**Решение:** Ждем результатов от Python-скрипта для анализа kernelcache iOS 17.3.1.

### 3. IOSurface может не работать напрямую
Функция `create_iosurface_rw()` пытается создать поверхность с указанием физического адреса, но это может быть заблокировано в iOS 17.3.1.
**Решение:** Альтернативный метод через IOMemoryDescriptor или использование уязвимости в другом драйвере.

### 4. PAC на ARM64e
Функция `strip_pac()` использует маску `0x0000000FFFFFFFFF`, но для некоторых указателей могут потребоваться другие методы очистки (например, через системные вызовы).

## 📋 Следующие шаги

1. **Запустить Python-скрипт** для анализа `iPhoneOS17.3.1.kernelcache`:
   - Найти точные адреса `_allproc`, `_current_task`
   - Найти адреса `amfi_allow_all`, `sandbox_enabled`
   - Получить паттерны для ключевых функций

2. **Интегрировать реальный эксплойт**:
   - Скопировать рабочий код из `third_party/darksword-kexploit-fun`
   - Или написать собственный на основе известных уязвимостей (IOSurface race condition)

3. **Протестировать на устройстве**:
   - Собрать IPA через `build_ipa_wsl.sh`
   - Установить на iPad8,9 с iOS 17.3.1
   - Проверить логи на наличие паник

4. **Добавить обработку ошибок**:
   - Если эксплойт падает, попробовать альтернативный метод
   - Добавить retry logic для всех критических операций

## 🔧 Технические детали

### Структура проекта
```
jbdc/
├── kexploit/
│   ├── darksword.m/h       # Эксплойт + примитивы
│   ├── shadow_pages.m/h    # Обход PPL
│   └── darksword_exploit.c # Обертка эксплойта
├── utils/
│   ├── offset_finder.m/h   # Поиск оффсетов
│   └── LaraPanicGuard.m/h  # Защита от паники
├── kernel/
│   ├── kernel_patcher.m/h  # Патчи ядра
└── LaraBootstrap.m         # Точка входа
```

### Зависимости
- IOSurface.framework (динамическая загрузка)
- IOKit.framework
- libxpf.a / libgrabkernel2.a (статические библиотеки)
- zstd (для распаковки bootstrap)

### Совместимость
- Устройство: iPad8,9 (A12, arm64e)
- iOS: 17.3.1
- Тип: Rootless (без монтирования системных разделов)
